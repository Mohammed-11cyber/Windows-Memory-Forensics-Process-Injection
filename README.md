# Windows Memory Diagnostics: Investigating Process Injection and Stealth C2

## Project Description
This incident response investigation focuses on low-level memory forensics and process lineage analysis to uncover an active fileless evasion compromise. Utilizing volatile memory dumps, I mapped the running system state to track process hierarchy anomalies, isolate malicious memory space hollowed out inside legitimate Windows binaries, and trace active outbound network communication channels back to known adversarial Command and Control infrastructure.

## Volatile Telemetry and Memory Structures Correlated
* **Process Tree Analysis:** Baselined standard Windows initialization structures against runtime states to identify illegitimate Parent-Child process relationships.
* **Memory Space Auditing (Malfind):** Scanned memory pages to isolate unauthorized protection flag modifications allowing executable code injection.
* **Network Object Tracking:** Correlated individual Process Identifiers directly to raw network sockets to map memory actions to network egress telemetry.

## Reconstructed Forensic Attack Lifecycle

### 1. Process Tree Lineage Anomaly
The standard Windows initialization chain (`smss.exe` -> `wininit.exe` -> `services.exe` -> `svchost.exe`) was validated across core processes. However, a critical hierarchy violation was flagged when a standard user text utility, `notepad.exe` (PID 2550), was captured spawning an essential system utility framework, `svchost.exe` (PID 3880). Legitimate `svchost.exe` processes must strictly be generated by the corporate Service Control Manager (`services.exe`), establishing this runtime branch as entirely illegitimate and indicative of a process hollowing attempt.

### 2. Executable Memory Injection (PAGE_EXECUTE_READWRITE)
Deep-dive memory space analysis confirmed active fileless injection inside the context of the rogue system framework. Memory pages allocated to `svchost.exe` (PID 3880) were flagged with explicit `PAGE_EXECUTE_READWRITE` (RWX) protection rights, exposing a volatile environment altered to execute dynamic machine code. Further parsing of the raw hex dump exposed an active MZ header layout inside the unmapped page space, validating the presence of a completely standalone executable running directly out of volatile system memory without a physical file footprint.

### 3. Concealed Network C2 Communication
Following successful memory injection, the compromised process context established active network outbounds. Socket correlation linked `svchost.exe` (PID 3880) directly to an active `ESTABLISHED` state session over destination IP address: `185.112.55.20`. The connection was processed entirely over Port 443 (HTTPS), demonstrating an operational tactic to mask malicious command-and-control metadata inside legitimate, encrypted enterprise web traffic.

## Forensic Conclusions
The investigation verified a successful, multi-stage process injection attack framework utilizing native Windows architecture for concealment. By hiding malicious execution paths directly within standard, trusted system processes and running entirely within volatile RAM, the adversary successfully bypassed common disk-level endpoint scanners. This highlights why modern SOC teams must complement traditional scanning with runtime memory inspection pipelines.
